### 1. หน้าหลัก — รายการสูตรอาหาร (`HomeScreen`)

> 📸 **ภาพหน้าจอ**
>
> | มุมมอง Grid | มุมมอง List |
> |---|---|
> | [screenshots](screenshots/Screenshot_1773644992.png) | _(วางภาพหน้าจอที่นี่)_ |

**ฟังก์ชันที่รองรับ:**

- **แสดงรายการสูตรอาหาร** — ดึงข้อมูลทั้งหมดจากฐานข้อมูล SQLite และแสดงผล
- **สลับมุมมอง Grid / List** — ปุ่มสลับมุมมองระหว่างแบบตาราง 2 คอลัมน์ และแบบรายการ
- **ค้นหาสูตรอาหาร** — กรองแบบ real-time จากชื่อ, หมวดหมู่ และส่วนผสม

- **Pull to Refresh** — ดึงข้อมูลใหม่โดยการเลื่อนลง

### 7. ฟังก์ชันระบบ

- **Debug Logger** — พิมพ์ตารางฐานข้อมูลทั้งหมดลง terminal ทุกครั้งที่มีการเขียนข้อมูล (INSERT, UPDATE, DELETE, TOGGLE) เฉพาะใน debug mode รองรับทั้งตาราง `recipes` และ `categories`
- **Database Migration** — รองรับการอัปเกรดโครงสร้างฐานข้อมูลอัตโนมัติ (v1→v2: เพิ่มคอลัมน์ `createdAt`; v2→v3: สร้างตาราง `categories`) โดยไม่สูญเสียข้อมูลเดิม
- **Sample Data** — แทรกสูตรอาหารตัวอย่าง 3 รายการ และหมวดหมู่เริ่มต้น 10 รายการเมื่อสร้างฐานข้อมูลครั้งแรก

## 🗄️ โครงสร้างฐานข้อมูล

### ตาราง `recipes`

| คอลัมน์ | ชนิดข้อมูล | คำอธิบาย |
|---|---|---|
| `id` | `INTEGER` PK | รหัสสูตรอาหาร (กำหนดอัตโนมัติ) |
| `title` | `TEXT` NOT NULL | ชื่อสูตรอาหาร |
| `category` | `TEXT` NOT NULL | หมวดหมู่ เช่น Pasta, Asian, Breakfast |
| `ingredients` | `TEXT` NOT NULL | ส่วนผสม (คั่นด้วย newline `\n`) |
| `instructions` | `TEXT` NOT NULL | วิธีทำ (คั่นด้วย newline `\n`) |
| `isFavorite` | `INTEGER` DEFAULT 0 | รายการโปรด (0 = ไม่ใช่, 1 = ใช่) |
| `cookTime` | `TEXT` | เวลาที่ใช้ทำ เช่น "30 mins" (optional) |
| `servings` | `TEXT` | จำนวนที่เสิร์ฟ เช่น "4" (optional) |
| `createdAt` | `TEXT` | วันที่บันทึก รูปแบบ ISO-8601 เช่น "2026-03-16" (optional) |

### ตาราง `categories`

| คอลัมน์ | ชนิดข้อมูล | คำอธิบาย |
|---|---|---|
| `id` | `INTEGER` PK | รหัสหมวดหมู่ (กำหนดอัตโนมัติ) |
| `name` | `TEXT` NOT NULL UNIQUE | ชื่อหมวดหมู่ (ห้ามซ้ำ) |

### ER Diagram (แบบง่าย)

```
┌─────────────────────────────────────────────┐
│                   recipes                   │
├────────────────┬────────────┬───────────────┤
│ PK  id         │  INTEGER   │ Auto Increment│
├────────────────┼────────────┼───────────────┤
│     title      │  TEXT      │ NOT NULL      │
│     category   │  TEXT      │ NOT NULL      │
│  ingredients   │  TEXT      │ NOT NULL      │
│  instructions  │  TEXT      │ NOT NULL      │
│   isFavorite   │  INTEGER   │ DEFAULT 0     │
│    cookTime    │  TEXT      │ Nullable      │
│    servings    │  TEXT      │ Nullable      │
│    createdAt   │  TEXT      │ Nullable      │
└────────────────┴────────────┴───────────────┘
             category (text)
                   │  (soft reference — ไม่ใช่ Foreign Key)
                   ▼
┌───────────────────────────────┐
│           categories          │
├──────────┬──────────┬─────────┤
│ PK  id   │ INTEGER  │ Auto   │
├──────────┼──────────┼─────────┤
│   name   │  TEXT    │ UNIQUE  │
└──────────┴──────────┴─────────┘

หมายเหตุ: recipes.category เก็บชื่อหมวดหมู่เป็น plain text
          ไม่ใช่ Foreign Key — หากลบหมวดหมู่ที่ยังมีสูตรใช้งาน
          ระบบจะแสดงข้อผิดพลาดและป้องกันการลบ
```

### หมายเหตุ

- ฐานข้อมูลจะถูกสร้างอัตโนมัติพร้อมสูตรอาหารตัวอย่าง 3 รายการในการรันครั้งแรก
- หากเคยติดตั้งเวอร์ชันเก่า ระบบจะ migrate ฐานข้อมูลอัตโนมัติ (v1→v2: เพิ่ม `createdAt`; v2→v3: เพิ่มตาราง `categories`) โดยไม่สูญเสียข้อมูลเดิม
- ข้อมูลทั้งหมดเก็บในเครื่อง ไม่มีการเชื่อมต่ออินเทอร์เน็ต
